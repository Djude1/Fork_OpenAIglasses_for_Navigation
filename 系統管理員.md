## 1. 系統需求

| 項目 | 需求 |
|------|------|
| Python | 3.11（嚴格限制） |
| 套件管理 | uv（推薦）或 pip |
| GPU | NVIDIA CUDA 11.8+（推薦；無 GPU 可用 CPU 但效能較差） |
| 硬體裝置 | Seeed Studio XIAO ESP32S3（或相容 ESP32 板） |
| API 帳號 | 阿里雲 DashScope（必填） |

## 2. 環境設定

### 2-1. 安裝依賴套件

```bash
# 建立虛擬環境（第一次使用）
uv venv

# 安裝所有依賴
uv sync
```

> 若需要 GPU 支援（CUDA 11.8），先單獨安裝 PyTorch：
> ```bash
> uv run pip install torch==2.0.1+cu118 torchvision==0.15.2+cu118 --index-url https://download.pytorch.org/whl/cu118
> ```

### 2-2. 建立 .env 設定檔

在專案根目錄建立 `.env` 檔案：

```env
# 必填：阿里雲 DashScope API 金鑰（用於語音辨識與 Qwen-Omni 對話）
DASHSCOPE_API_KEY=sk-xxxxx

# 選填：OpenRouter API 金鑰（用於 Nemotron 視覺描述）
OPENROUTER_API_KEY=sk-or-xxxxx

# 選填：Google Gemini API 金鑰（視覺備援）
GEMINI_API_KEY=xxxxx

# 可選覆寫參數（下方為預設值，不需要修改可不填）
BLIND_PATH_MODEL=model/yolo-seg.pt
OBSTACLE_MODEL=model/yoloe-11l-seg.pt
YOLOE_MODEL_PATH=model/yoloe-11l-seg.pt
TRAFFICLIGHT_MODEL=model/trafficlight.pt
SHOPPING_MODEL=model/shoppingbest5.pt
AIGLASS_MASK_MIN_AREA=1500
TTS_INTERVAL_SEC=1.0
ENABLE_TTS=true
SERVER_PORT=8081
```

## 4. 啟動服務

### 4-1. 使用 ESP32 硬體（正式部署）

```bash
uv run python app_main.py
```

伺服器成功啟動後，終端機會顯示：

```
INFO:     Uvicorn running on http://0.0.0.0:8081
[NAVIGATION] 模型文件存在，开始加载...
[AUDIO] 音频系统初始化完成（预加载+工作线程）
```

> 若顯示 `未設定 DASHSCOPE_API_KEY` 錯誤，請確認 `.env` 已正確設定。

### 4-2. 使用電腦測試（無 ESP32）

需要開兩個終端機，**順序不可對調**：

**終端機 1 — 主伺服器**（先啟動）
```bash
uv run python app_main.py
```

**終端機 2 — ESP32 模擬器**（後啟動）
```bash
uv run python esp32_simulator.py
```

模擬器以電腦的攝影機、麥克風、喇叭取代 ESP32 硬體，功能與實機相同。

## 5. 硬體連接（ESP32）

### 5-1. 燒錄韌體

使用 Arduino IDE 開啟 `compile/compile.ino`，選擇板子為 **Seeed Studio XIAO ESP32S3**，燒錄至裝置。

> 若使用其他 ESP32 板，請參考 `compile/camera_pins.h` 調整腳位設定。

### 5-2. 設定 Wi-Fi 與伺服器位址

在 `compile/compile.ino` 中找到以下區段，填入實際的 Wi-Fi 與伺服器資訊：

```cpp
const char* ssid = "你的WiFi名稱";
const char* password = "你的WiFi密碼";
const char* server_host = "192.168.x.x";  // Python 伺服器的 IP
const int server_port = 8081;
```

### 5-3. 硬體接線（XIAO ESP32S3）

| 元件 | 接腳 |
|------|------|
| 攝影機（DVP） | XCLK=10, SIOD=40, SIOC=39 |
| 麥克風（PDM） | CLK=42, DAT=41 |
| 喇叭 MAX98357A（I2S） | BCLK=7, LRCK=8, DIN=9 |
| IMU ICM42688（SPI） | SCK=D0, MOSI=D1, MISO=D2, CS=D3 |

### 5-4. 連接方式

ESP32 開機後會自動透過 WebSocket 連線至伺服器：

| 資料類型 | 傳輸方式 | 端點 |
|----------|----------|------|
| 視訊影像 | WebSocket | `ws://<IP>:8081/ws/camera` |
| 麥克風音訊 | WebSocket | `ws://<IP>:8081/ws_audio` |
| IMU 姿態感測 | UDP | Port `12345` |
| 喇叭音訊下行 | HTTP | `http://<IP>:8081/stream.wav` |

## 7. 功能總覽與語音指令

所有功能皆透過**語音指令**觸發。ESP32 麥克風收音後，由 DashScope ASR 辨識，再由系統判斷執行對應功能。

### 7-1. 盲道導航

沿著地面盲道（導盲磚）行走，系統自動辨識盲道位置並語音引導方向。

| 語音指令 | 說明 |
|----------|------|
| `開始導航` / `盲道導航` / `幫我導航` | 啟動盲道導航模式 |
| `停止導航` / `結束導航` | 停止導航 |

系統會持續語音播報：
- **直行**：「請繼續直行。」
- **偏移修正**：「請向左平移。」/ 「請向右平移。」
- **轉彎**：「請向左轉。」/ 「請向右轉。」
- **前方障礙物**：「前方有障礙物，注意避讓。」

### 7-2. 過馬路輔助

偵測斑馬線位置與紅綠燈狀態，引導使用者安全過馬路。

| 語音指令 | 說明 |
|----------|------|
| `開始過馬路` / `幫我過馬路` | 啟動過馬路模式 |
| `過馬路結束` / `結束過馬路` | 停止過馬路模式 |
| `立即通過` / `現在通過` / `繼續` | 手動確認通過（等待紅綠燈時） |

過馬路流程：
1. 尋找斑馬線 → 2. 等待綠燈 → 3. 引導通過 → 4. 尋找下一段盲道

### 7-3. 紅綠燈偵測

單獨啟動紅綠燈偵測模式，不進行完整過馬路流程。

| 語音指令 | 說明 |
|----------|------|
| `檢測紅綠燈` / `看紅綠燈` | 啟動紅綠燈偵測 |
| `停止檢測` / `停止紅綠燈` | 停止偵測 |

### 7-4. 物品尋找

使用 YOLO 開放詞彙偵測，協助找到指定物品。

| 語音指令 | 說明 |
|----------|------|
| `找一下[物品名稱]` | 開始尋找指定物品，例如「找一下水杯」 |
| `找到了` / `拿到了` | 停止尋找，若之前在導航則自動恢復導航 |

> 系統會透過 Qwen-Turbo 將中文物品名稱轉換為英文標籤，再傳入 YOLO 進行偵測。

### 7-5. 視覺查詢（場景描述）

請 AI 描述當前攝影機畫面中的內容。

| 語音指令 | 說明 |
|----------|------|
| `幫我看` / `幫我看下` | 描述眼前場景 |
| `看看` / `識別一下` | 識別畫面中的物體 |
| `那是什麼` / `前面是什麼` | 詢問前方物體 |
| `前面有什麼` / `前方有什麼` | 詢問前方情況 |
| `前方情況` / `幫我描述` / `描述一下` | 描述當前環境 |

> 系統優先使用 **Nemotron 12B VLM** 回答（每次查詢間隔至少 3 秒），若失敗則降級至 Qwen-Omni。

### 7-6. 自由對話（Qwen-Omni）

不符合以上特定指令的語音，會進入 Qwen-Omni 多模態 AI 對話模式，可結合當前攝影機畫面進行問答。

> 在導航或紅綠燈偵測模式中，只有包含特定關鍵字（如「幫我看」、「找一下」）的語音才會進入對話，避免誤觸發。

### 7-7. 打斷 AI 說話

| 語音指令 | 說明 |
|----------|------|
| `停下` | 立即停止 AI 語音播放 |
| `別說了` | 立即停止 AI 語音播放 |
| `停止` | 立即停止 AI 語音播放 |

## 8. AI 呼叫說明

系統整合了多個 AI 服務，依功能自動選擇：

| AI 服務 | 用途 | 需要的金鑰 |
|---------|------|------------|
| **DashScope ASR**（Paraformer） | 語音辨識（麥克風 → 文字） | `DASHSCOPE_API_KEY` |
| **Qwen-Omni-Turbo** | 多模態對話（文字 + 圖像 → 語音回覆） | `DASHSCOPE_API_KEY` |
| **Qwen-Turbo** | 中文物品名稱 → 英文 YOLO 標籤 | `DASHSCOPE_API_KEY` |
| **Nemotron Nano 12B VLM** | 視覺場景描述（優先用於視覺查詢） | `OPENROUTER_API_KEY` |
| **Google Gemini** | 視覺理解備援 | `GEMINI_API_KEY` |

### 呼叫流程

```
使用者說話
    │
    ▼
DashScope ASR（語音 → 文字）
    │
    ▼
關鍵字比對
    ├── 導航指令 ──────────────→ NavigationMaster 狀態機
    ├── 視覺查詢關鍵字 ────────→ Nemotron VLM（失敗則降級 Qwen-Omni）
    ├── 找物品（找一下...） ───→ Qwen-Turbo 提取標籤 → YOLO 偵測
    └── 其他自由對話 ──────────→ Qwen-Omni（含攝影機畫面）
```

## 10. 進階設定（.env 參數）

以下參數可在 `.env` 中覆寫預設值：

### 伺服器

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `SERVER_HOST` | `0.0.0.0` | 伺服器監聽位址 |
| `SERVER_PORT` | `8081` | 伺服器埠號 |
| `UDP_PORT` | `12345` | IMU 資料接收埠號 |

### 語音 / TTS

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `ENABLE_TTS` | `true` | 是否啟用語音播報 |
| `TTS_INTERVAL_SEC` | `1.0` | 相同語音最短間隔（秒） |
| `INTERRUPT_KEYWORDS` | `停下,別說了,停止` | 打斷熱詞（逗號分隔） |
| `OMNI_VOICE` | `Cherry` | Qwen-Omni TTS 聲音角色 |

### 盲道導航

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `AIGLASS_MASK_MIN_AREA` | `1500` | 盲道最小偵測面積（像素） |
| `AIGLASS_STRAIGHT_INTERVAL` | `4.0` | 直行播報間隔（秒） |
| `AIGLASS_DIRECTION_INTERVAL` | `3.0` | 轉向播報間隔（秒） |
| `AIGLASS_OBS_INTERVAL` | `15` | 障礙物偵測抽幀間隔（幀） |
| `AIGLASS_BLINDPATH_INTERVAL` | `8` | 盲道 YOLO 抽幀間隔（幀） |

### 過馬路

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `CROSSWALK_MIN_CONF` | `0.3` | 斑馬線最低置信度 |
| `CROSSWALK_MIN_AREA` | `5000` | 斑馬線最小面積（像素） |
| `CROSSWALK_ANGLE_THRESH_DEG` | `5.0` | 對齊角度容許誤差（度） |
| `CROSSWALK_OFFSET_THRESH` | `0.08` | 對齊偏移容許誤差（相對寬度） |

### GPU / 效能

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `AIGLASS_DEVICE` | `cuda:0` | 推理裝置（`cuda:0` 或 `cpu`） |
| `AIGLASS_AMP` | `bf16` | 混合精度策略（`bf16`/`fp16`/`off`） |
| `CONF_THRESHOLD` | `0.20` | YOLO 全域置信度門檻 |
| `NEMOTRON_MIN_INTERVAL` | `3.0` | Nemotron 請求最短間隔（秒） |
| `PERF_DEBUG` | `false` | 是否輸出效能偵錯資訊 |

### 麥克風靈敏度（收音最大化）

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `ASR_PCM_GAIN` | `8.0` | ESP32 麥克風增益倍數（建議 5~10，越高越靈敏但可能失真） |
| `ASR_STANDBY_RMS_THRESH` | `150.0` | 待機模式送出批次的最低音量門檻（越低越靈敏，0~32767） |

> 若喚醒詞「哈囉 曼波」常接收不到，先確認 `ASR_PCM_GAIN` 是否夠高，再降低 `ASR_STANDBY_RMS_THRESH`。

### 說話人聲紋驗證

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `SPEAKER_VERIFY_ENABLED` | `false` | 是否啟用說話人驗證（只有你的聲音能觸發喚醒詞） |
| `SPEAKER_EMBED_PATH` | `model/speaker_embed.pkl` | 聲紋儲存路徑 |
| `SPEAKER_THRESHOLD` | `0.82` | 相似度門檻（0~1，越高越嚴格；建議 0.75~0.88） |

> 啟用前必須先完成聲紋錄製，詳見第 12 節。

## 12. 說話人聲紋驗證（只接受你的聲音）

系統支援說話人驗證功能，錄製你的聲紋後，麥克風只會對你的聲音有反應，可有效過濾路人或環境噪音誤觸發喚醒詞。

### 12-1. 聲紋管理 Dashboard（推薦方式）

伺服器啟動後，直接在瀏覽器開啟：

```
http://localhost:8081/speaker
```

提供完整 GUI 操作，無需使用 curl：
- **聲紋錄製**：按下「開始錄製聲紋」，對麥克風說話 10 秒，頁面顯示倒數進度條與完成通知
- **功能開關**：說話人驗證 / 持續監測 / 旁路喚醒詞 各自有 Toggle 開關
- **驗證測試**：按下「開始 3 秒驗證測試」，即時顯示相似度分數與通過/失敗
- **即時儀表**：音量 RMS 條形圖、相似度量表（含門檻白線）
- **監測歷史**：每次持續監測事件自動新增到歷史表格
- **Debug 日誌**：即時顯示所有 SSE 推送事件與時間戳

> 主監控頁面右上角的「🎙 聲紋」按鈕可直接跳轉。

---

### 12-2. 錄製聲紋（一次性設定）

**GUI 方式（推薦）**：開啟 `http://localhost:8081/speaker`，點擊「開始錄製聲紋」。

**API 方式**：

```bash
# 步驟 1：開始收音（10 秒）
curl -X POST http://localhost:8081/api/enroll_speaker

# 步驟 2：查詢錄製結果
curl http://localhost:8081/api/enroll_status
```

錄製成功後 `"enrolled": true`，聲紋儲存於 `model/speaker_embed.pkl`。

**步驟**：啟動伺服器 → 確認 ESP32 / 模擬器已連線 → 對著麥克風正常說話 10 秒（越自然越準確）。

### 12-3. 啟用說話人驗證

在 `.env` 加入以下設定後重啟伺服器：

```env
SPEAKER_VERIFY_ENABLED=true
```

啟用後，待機模式（等待喚醒詞「哈囉 曼波」）只有與聲紋相似度超過門檻的聲音才會送往語音辨識。

> 也可在 Dashboard 直接切換 Toggle，不需重啟（效果為暫時開關，不修改 `.env`）。

### 12-4. 調整相似度門檻

```env
SPEAKER_THRESHOLD=0.82   # 預設（建議值）
```

| 門檻值 | 適用情境 |
|--------|----------|
| `0.75` | 寬鬆，允許聲音相近的人觸發 |
| `0.82` | 適中（預設），一般環境 |
| `0.88` | 嚴格，噪音環境仍只接受本人 |

用 Dashboard 的「持續監測」功能觀察相似度分佈，再調整此門檻：
- 本人說話：相似度通常 `0.85~0.95`
- 陌生人說話：相似度通常低於 `0.75`

### 12-5. 管理聲紋（API 參考）

| 動作 | API |
|------|-----|
| 重新錄製聲紋 | `POST /api/enroll_speaker`（覆蓋舊聲紋） |
| 查詢目前狀態 | `GET /api/enroll_status` |
| 刪除聲紋（停用驗證） | `POST /api/delete_speaker` |

| 關閉持續監測 | `POST /api/verify_continuous?enabled=false` |

### 12-6. 注意事項

- 聲紋只在**待機模式**（等待喚醒詞）過濾，進入主動錄音後不再驗證（避免干擾指令接收）
- 使用純 numpy MFCC 演算法，不需額外 GPU 資源
- 錄製環境盡量安靜，避免大量背景噪音影響聲紋品質
- 若更換麥克風或使用環境大幅改變，建議重新錄製聲紋

## 常見問題

**Q: 啟動時出現 `未設定 DASHSCOPE_API_KEY`**
A: 確認 `.env` 檔案存在於專案根目錄，且 `DASHSCOPE_API_KEY` 已正確填入。

**Q: 模型載入失敗**
A: 確認 `model/` 目錄下有所有必要的 `.pt` 與 `.task` 檔案（參考第 3 節）。

**Q: ESP32 無法連線**
A: 確認 ESP32 韌體中的 Wi-Fi 名稱、密碼、伺服器 IP 是否正確，且 ESP32 與伺服器在同一網路下。

**Q: 語音辨識沒有反應**
A: 確認 `DASHSCOPE_API_KEY` 有效，且 ESP32 麥克風已正確接線（PDM CLK=42, DAT=41）。

**Q: 視覺查詢（看看/幫我看）沒有回應**
A: 確認 `OPENROUTER_API_KEY` 已設定。若未設定則會降級使用 Qwen-Omni（需要 `DASHSCOPE_API_KEY`）。

**Q: 想在沒有 ESP32 的情況下測試**
A: 系統在沒有 ESP32 連線時仍可啟動，可透過監控頁面的上傳功能或修改程式碼使用本機攝影機（`CAM_INDEX=0`）進行測試。
